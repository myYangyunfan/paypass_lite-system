## 准备工作：安装 VS Code

### 第一步：下载 VS Code

在浏览器中打开以下网址：

```
https://code.visualstudio.com/
```

页面打开后，浏览器会自动识别你的操作系统（Windows），你应该会看到一个大的蓝色按钮写着 **"Download for Windows"**。

点击这个按钮，会下载一个后缀为 `.exe` 的安装包到你的电脑（通常是浏览器下载文件夹里）。

### 第二步：安装 VS Code

找到你下载的 `VSCodeSetup-xxx.exe` 文件（xxx 是版本号），双击它开始安装。

安装向导里，全部选**默认选项**，不用改，直接一路点 **"下一步"**，最后点 **"安装"** 即可。

安装完成后，VS Code 会自动打开。

### 第三步：安装 Git（必须）

> Git 是 VS Code 连接 Gitee 的桥梁，必须先装

1. 打开浏览器，访问：https://git-scm.com/download/win
2. 页面会弹出下载框，或者你点击 **"Click here to download"**
3. 下载 `Git-xxx-64-bit.exe` 文件
4. 双击安装，同样一路点 **"Next"**，最后 **"Install"**
5. 安装完成后会打开一个黑色的命令行窗口，显示安装成功，按回车关掉

**验证 Git 是否安装成功：**

1. 在电脑左下角搜索框输入 `cmd`，回车，打开命令提示符
2. 输入下面这行字（复制粘贴进去），回车：

```
git --version
```

3. 如果显示类似 `git version 2.x.x` 的字样，说明安装成功了

## 第一步：注册 Gitee 账号

### 1.1 打开 Gitee

在浏览器地址栏输入：

```
https://gitee.com/
```

回车，进入 Gitee 首页。

### 1.2 点击注册

在页面右上角，找到 **"注册"** 按钮并点击。

### 1.3 填写注册信息

需要填写的内容：
- **手机号**：输入你自己的手机号
- **验证码**：点击获取验证码，输入手机收到的短信验证码
- **设置密码**：设置一个你记得住的密码
- **昵称**：给自己起一个名字（团队成员之间不要重名）

填写完后，勾选"我已阅读并同意"相关协议，点击 **"注册"**。

> 建议：所有人的 Gitee 昵称格式统一，比如 `yang-dev`、`zhao-dev`、`luo-dev`、`lin-dev`、`chen-dev`，方便识别

### 1.4 邮箱验证（如果提示）

注册成功后，Gitee 可能要求你验证邮箱。去你的邮箱里找 Gitee 发来的验证邮件，点击邮件里的验证链接即可。

## 第二步：在 Gitee 上创建仓库

> 提示：这一步只需要**一个人**（通常是队长）做一次，其他队员不需要重复做

### 2.1 创建新仓库

登录 Gitee 后，在页面右上角点击 **"+"** 号，在下拉菜单中选择 **"新建仓库"**。

### 2.2 填写仓库信息

| 字段 | 填写内容 |
|------|---------|
| **仓库名称** | `paypass-lite` |
| **路径** | 自动生成，会显示为 `paypass-lite`，不用改 |
| **归属** | 选择你的个人仓库 |
| **仓库公开性** | 选择 **"公开"**（公开仓库免费） |
| **初始化仓库** | **不要勾选**"使用README初始化仓库"（因为我们要自己上传文件） |
| **许可证** | 不选（保持空白） |
| **私有化** | 不要勾选 |

填写好后，点击 **"创建"**。

### 2.3 仓库创建完成

创建成功后，你会看到一个页面，里面有一行地址，类似：

```
https://gitee.com/你的用户名/paypass-lite.git
```

把这个地址**复制**下来（鼠标选中，然后 Ctrl+C），发给你的队员。

> **队长需要做的：**
> 把这个仓库地址发给每一个队员，然后让他们按照下面的教程操作

### 2.4 把队员加入仓库（可选，推荐做）

在仓库页面，点击右上角 **"管理"** → 左侧菜单选 **"成员管理"** → 点击 **"添加成员"** → 输入队员的 Gitee 昵称或手机号 → 选择权限为 **"开发者"** → 点击确认。

这样所有队员都可以往这个仓库推送代码了。

在搜索框里输入：

### 需要记住的三个图标（重点）

| 图标 | 名称 | 作用 |

## 第五步：每日上传流程（纯图形化）

> **重要原则：只上传自己负责的文件，不要碰别人负责的文件！**

---

### 👤 杨 的每日操作

**第一步：打开 VS Code 并打开仓库文件夹**

双击 VS Code 图标 → 点击左上角 **"文件"** → **"打开文件夹"** → 选择 `paypass-lite` 文件夹。

**第二步：在左侧文件列表中找到你今天要上传的文件**

按下面的计划表找到对应的文件（如果文件不存在，需要新建）。

点击 **左侧第一个 📁 图标** 展开文件夹，找到目标文件。

**第三步：双击打开文件，粘贴代码，保存**

如果文件不存在：在文件资源管理器里，右键点击目标文件夹 → **"新建文件"** → 输入文件名（如 `hashtable.h`）→ 双击打开 → 把代码粘贴进去 → 按 `Ctrl+S` 保存。

**第四步：添加到暂存区**

按 `Ctrl+S` 保存后，按 **Ctrl+Shift+G** 打开源代码管理面板（或者直接点击左侧第三个 🔀 图标）。

你应该能看到你的文件出现在列表里，前面标记为 **U**（新增文件）。

找到你今天要上传的文件，鼠标悬停在这一行上，会出现一个 **"+"** 号。**点击这个 "+" 号**。

> 这一步的作用是：告诉 Git"我准备好了，这个文件要提交了"

点击后，文件会从"Untracked"移到"已暂存"区域（Staged）。

**第五步：填写提交信息**

在源代码管理面板顶部，有一个输入框写着 *"消息（按 Ctrl+Enter 提交）"*。

在这个框里**输入你的提交信息**（参考下面的计划表）：

```
[core] add: 哈希表头文件，定义HashTable结构体和djb2哈希函数
```

**第六步：提交**

输入完提交信息后，按键盘上的 **Ctrl+Enter** 提交。

> 提交（Commit）的意思是把文件保存到本地的历史记录里

提交成功后，这个文件会从暂存区消失（表示已经保存好了）。

**第七步：推送到远程**

提交成功后，VS Code 右下角会弹出提示：*"是否将变更同步到远程？"*

点击 **"同步"** 或 **"Sync"** 按钮（也可能是向上和向下两个箭头的图标）。

> 这一步是把本地保存的记录上传到 Gitee 云端

如果弹出登录框，输入你的 Gitee 用户名和密码。

等待几秒，右下角会显示同步完成的提示（"同步成功"或"All changes pushed"）。

**第1天完成！** ✅

## 第六步：验证上传是否成功

无论是谁上传完成后，都应该在 Gitee 上验证。

### 验证步骤：

1. 打开浏览器，访问你的仓库地址：

```
https://gitee.com/你的用户名/paypass-lite
```

2. 登录你的 Gitee 账号

3. 在仓库页面，找到并点击 **"提交"** 或 **"commits"** 链接（一般在页面中上部）

4. 在提交记录里，应该能看到你刚才的提交信息，比如：

```
[core] add: 哈希表头文件，定义HashTable结构体和djb2哈希函数
```

5. 点击这条提交记录，可以看到具体上传了哪些文件

### 如果看不到提交记录怎么办？

- 检查一下是否所有步骤都完成了（尤其是最后的"同步"步骤）
- 可能需要刷新页面（按 F5）
- 如果同步时报错了，参考下面的常见问题

## 第七步：常见问题与解决方法

---

### 问题1：VS Code 提示"需要登录 Gitee"

**解决方法：**

1. 点击 VS Code 右下角的 🔔 通知，或者在弹出的登录框里
2. 点击 **"Sign In"** 或 **"登录"**
3. 选择 **"Gitee"** 或 **"在浏览器中登录"**
4. 浏览器会打开 Gitee 授权页面，点击 **"授权"**
5. 回到 VS Code，应该就登录成功了

---

### 问题2：VS Code 提示"未能推送，因为未能认证"

**原因：** 没有配置 Gitee 的用户名和邮箱

**解决方法：**

1. 在 VS Code 中按 F1，输入 `git config`，选择 **"Git: 配置用户名"**
2. 输入你的 Gitee 用户名（不是昵称，是登录名）
3. 再次按 F1，输入 `git config`，选择 **"Git: 配置电子邮件"**
4. 输入你注册 Gitee 时用的邮箱
5. 重新尝试推送

---

### 问题3：点击同步按钮后卡住不动

**解决方法：**

1. 等 30 秒，如果还没反应，按 `Esc` 键取消
2. 在 VS Code 底部状态栏查看是否有错误提示
3. 重新尝试：点击 🔀 → 再次点击 🔄 同步

---

### 问题4：文件旁边显示 M（已修改）而不是 U（新增）

**含义：** 这个文件之前已经被别人上传过了，现在你做了修改

**如果是这种情况：**

- **杨、赵**：可以正常提交修改（如果确实在修复问题）
- **罗、林、陈**：如果是误操作（不小心改了别人的文件），需要撤销：在文件上右键 → 选择 **"放弃更改"**

---

### 问题5：想撤销刚才的提交（还没推送）

**解决方法：**

1. 打开源代码管理面板
2. 在提交记录上方，找到 **"…"（更多操作）** 按钮（三个点）
3. 点击后选择 **"撤消上一次提交"**
4. 注意：撤销后文件会回到提交前的状态，需要重新提交

---

### 问题6：推送时报冲突错误

**原因：** 别人已经推送了新内容，你本地的版本太旧了

**解决方法：**

1. 在源代码管理面板，点击 **"…"** → **"拉取"**（Pull）
2. 拉取成功后，VS Code 会合并内容
3. 如果有冲突文件（标红），手动解决冲突（保留你需要的代码）
4. 然后重新添加 → 提交 → 推送

---

### 问题7：不小心删除了文件，怎么恢复？

**解决方法：**

1. 在源代码管理面板，找到 **"更改"** 列表
2. 找到被删除的文件，鼠标悬停，会出现 **"+"** 和 **"↩"** 按钮
3. 点击 **"↩"** 按钮（"放弃更改"），文件就会恢复到删除前的状态
